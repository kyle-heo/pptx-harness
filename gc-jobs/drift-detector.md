> PPT 생성 시 디자인 일관성 저하(드리프트)를 감지합니다.
> 트리거: 동일 P-번호 위반 2회 연속 / SIGNALS.md 누적 패턴 업데이트 시

---

## 감지 항목

1. **색상 드리프트**: 동일 PPT 내에서 팔레트 외 색상 사용
2. **폰트 드리프트**: 선택된 페어링 외 폰트 혼용
3. **레이아웃 드리프트**: 동일 레이아웃 3슬라이드 이상 연속 사용
4. **P-번호 반복 위반**: 동일 실수 패턴 누적

---

## 실행 지시사항 (Codex에게)

### 1단계: SIGNALS.md 누적 패턴 분석

`SIGNALS.md` > "누적 패턴 분석" 테이블 확인:
- 누적 위반 3회 이상인 P-번호 식별
- 해당 린터 심각도 `warn` → `error` 격상

### 2단계: design-system.md 갱신

반복 실수 패턴이 있으면 해당 섹션에 **경고 박스** 추가:
```markdown
> ⚠️ 반복 위반 패턴: [구체적 설명] (누적 N회)
> 반드시 이 패턴을 피하십시오.
```

### 3단계: AGENTS.md 갱신

```
| YYYY-MM-DD | drift-detector: [P-번호] 누적 N회 → 린터 강도 격상 | 자동 |
```
